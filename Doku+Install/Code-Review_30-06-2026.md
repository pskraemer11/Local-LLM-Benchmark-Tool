## 5. Schnittstellen-Analyse

### 5.1 run_benchmarks_v10.py -> custom_benchmark_v10.py

```
Run-Benchmark uebergibt:
  --model-key    (z.B. "qwen3-coder-reap-25b-a3b-i1@iq4_xs")
  --api-model    (z.B. "qwen3-coder-reap-25b-a3b-i1@iq4_xs@q6_k")
  --sample-size  (int)
  --benchmark    ("DS1000" oder "CoderEval")
  --qwen-prompt  (optional, nur bei qwen3.5)

custom_benchmark_v10.py erwartet:
  --non-interactive  (immer gesetzt)
  --model-key        -> MODEL_CONFIG Lookup
  --api-model        -> API-Call (MUSS mit lms ps übereinstimmen)
```

**Risiko:** Wenn `api_model` nicht der exakten Load-ID entspricht, antwortet LM Studio mit HTTP 400. Das wird durch `model_info["_api_model"]` aus `lms ps --json` sichergestellt.

### 5.2 run_benchmarks_v10.py -> lm_eval CLI

```
model_args = "base_url=...,model={api_model},num_concurrent=1"
  + optionale Parameter aus _get_lmeval_params():
    - max_tokens, temperature, top_p, top_k, min_p, until

WICHTIG: Der model-Parameter MUSS der exakten Load-ID entsprechen.
  Beispiel: "model=qwen3-coder-reap-25b-a3b-i1@iq4_xs@q6_k"
  Falsch:   "model=qwen3-coder-reap-25b" (unvollstaendig -> 400)
  Falsch:   "model=check" (ungueltig -> haengt)
```

### 5.3 csv_writer.py -> consolidate_results_v10.py

```
csv_writer schreibt:
  tasks_*.csv    -> FIELDNAMES = TASK_FIELDS (Per-Task)
  model_*.csv    -> FIELDNAMES = MODEL_FIELDS (aggregiert)
  modell_*.csv   -> FIELDNAMES = SUMMARY_FIELDS (akkumuliert)
  konsolidiert_aktuell.csv -> CONSOLIDATED_FIELDS

consolidate_results_v10.py liest:
  read_custom_csv()      -> erwartet "score", "tokens_per_sec", "latency_s",
                            "cpu_during"/"cpu_pct", "gpu_during"/"gpu_pct"
  try_read_evalplus()    -> liest evalplus JSON-Dateien
  read_lmeval_per_model() -> liest lm_eval results_*.json
  read_agentic()         -> liest tool_eval_bench JSON
```

**Konsistenz-Problem:** `read_custom_csv()` sucht nach `cpu_during`/`gpu_during` (Per-Task-Felder), aber die Summary-CSVs haben `cpu_pct`/`gpu_pct`. Das ist korrekt, da `read_custom_csv()` nur fuer Per-Task-CSVs (tasks_*.csv) gedacht ist.

### 5.4 benchmark_config.py -> alle Skripte

```
DISPLAY_NAMES  -> consolidate_results_v10.py (Lookup), run_all_dense.py (veraltet)
WHITELIST      -> consolidate_results_v10.py (Iteriert ueber alle Modelle)
QUANT_MAP      -> consolidate_results_v10.py (_lookup_vram)
CAT_WEIGHTS    -> consolidate_results_v10.py (compute_category_scores)
OVERALL_WEIGHTS -> consolidate_results_v10.py (compute_category_scores)
PIPELINE_TIMEOUTS -> model_manager.py (importiert), run_benchmarks_v10.py
```
