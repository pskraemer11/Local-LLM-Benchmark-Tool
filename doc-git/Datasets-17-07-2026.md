# Benchmark Datasets – Sources & References (as of 30.06.2026)

This project uses standardized benchmark datasets for evaluating local LLMs.

All test data (questions and answers) are obtained via `download_real_benchmarks.py` and stored as
JSONL files in the `simple_evals/` directory.

**Since v21 (23.06.2026):** The overall score weighting focuses on **Coding + Math**.

**Since v10 (30.06.2026):** All three pipelines run via `run_benchmarks_v13.py` as launcher:
- Custom Benchmarks (DS1000, CoderEval) via `custom_benchmark_v13.py`.
- LM-Eval Tasks (MathQA, ARC, HellaSwag, TruthfulQA, MMLU-Pro) via `lm_eval` CLI.
- EvalPlus Tasks (HumanEval+, MBPP+) via `evalplus` CLI. Agentic via `tool_eval_bench` CLI.

**Selection Procedure / Sampling:**
The number of questions/tasks in the benchmarks is sometimes very large, which would result in extremely long runtimes
on a PC, even with an NVIDIA graphics card.
Therefore, a smaller subset (sample) of questions is drawn **randomly** from the population.
The sample size `SampleSize` (default: 50) can be passed via `--sample-size N`.

The seed of the random generator can also be set via option, see
`run_benchmarks_v13.py -help`.

For benchmarks with thematic subgroups (subsets, libraries, subject areas), `ceil(SampleSize / NumGroups)`
questions are drawn per group, so that each group is equally represented.

Running the benchmark script `custom_benchmark_v13.py` **no longer** requires internet access –
only for the preceding download of test data.

## Appendix C: ARC – Easy vs. Challenge

The ARC dataset (AI2 Reasoning Challenge) has two difficulty levels:

| Split | Questions | Difficulty | Typical tasks |
|-------|-----------|------------|---------------|
| **ARC-Easy** | 5197 | Easy | Question-answer with pure recall knowledge (e.g. "How many bones does the human body have?") |
| **ARC-Challenge** | 2590 | Hard | Solvable only by combining multiple knowledge facts |

We use exclusively **ARC-Challenge** (test split), since Easy is too trivial for modern LLMs.

## Appendix D: TruthfulQA – MC1 vs. MC2

TruthfulQA tests the tendency of LLMs to repeat false statements ("hallucinations").
There are two evaluation formats:

| Format | Meaning | Selection |
|--------|---------|----------|
| **MC1** (Multiple Choice 1) | **Exactly one** answer is correct. The label field contains a list with exactly one `1` (correct) and otherwise `0` (incorrect). | Used – unambiguously evaluable |
| **MC2** (Multiple Choice 2) | **Multiple** answers can be correct. Evaluation is done via probabilities (Multiple True). | Not used – complex evaluation |

We use **MC1** and skip tasks without a clear `1` label (erroneous entries).

## Appendix E: HellaSwag – Label 0–3

Each HellaSwag task consists of a context (situation) and 4 possible endings.
The `label` field indicates which ending is correct:

| Label | Answer | Meaning |
|-------|--------|---------|
| `"0"` | A | The first ending is correct |
| `"1"` | B | The second ending is correct |
| `"2"` | C | The third ending is correct |
| `"3"` | D | The fourth ending is correct |

Tasks with `label` outside 0–3 or an empty label are skipped.
The 4 endings come from a crowdsourcing study: three endings are human-written
"traps" (plausible but incorrect), one ending is the actual continuation.
